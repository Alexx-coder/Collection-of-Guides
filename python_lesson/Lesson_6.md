#### Работа с файлами используя `with open`

- Наверное вы помните мой пример с open, но без with, так в чем разница?

- Используя `with open` будет автоматическое закрытие файла и это удобно, без `with` надо будет в ручную, а это неудобно...

- Давайте разберём конструкцию:

```python

with open(файл, 'работа_с_файлом') as f:
    f.работа_с_файлом_функция()

```

- Вам наверное, не понятно, но давайте разберём!

- Переменная `файл` - это имя файл, например, README.md

- а вот `'работа_с_файлом'`- это вид работы, например: запись, чтение, дозапись

- Запись это: 'w', а если там бинарная запись: 'wb'

- Чтение это: 'r', а если там бинарное чтение: 'rb'

- Дозапись это: 'a', а если там бинарная дозапись: 'ab'

- Да, это сокращенно, w - write, r - read, a - append, b - bytes

- Теперь разберём `работа_с_файлом_функция()` - это функция, для запись - write(содерижимое), чтения - read(), дозаписи - write(содержимое), создания - write('')

- Да почему для создания файла - write(''), пустые ковычки для записи пустого файла, чтобы не произошла потом ошибка записи

- Теперь попробуйте:

```python

with open("file.txt", "w") as f:
    f.write("Привет!") # Создатся файл в вашей текущей папке 

with open("file.txt", "r") as r:
    contents = r.read()
    print(contents) # Вывод: содержимое файлы file.txt - Привет!

with open("file.txt", "a") as q:
    q.write("Как дела!") # Запишет Как дела!

with open("file.txt", "r") as w:
    contents = w.read()
    print(contents) # Вывод: Привет! ; Как дела!

```

- Если вы всё таки без `with` - не забудьте про `close()`

- Совет: если вы хотите дописать используйте 'a', а то оно перезапишет содержимое

- Готово! Вы знаете как работать с файлами, но еще много что изучать

#### Низкоуровневая работа с файлами

- Теперь мы будем работать с системой

- Давайте проверим существует ли файл:

```python
import os

if os.path.exists("file.txt"):
    print("[+] Файл существует!")

else:
    print("[-] Файл не существует!")

```
- А теперь мы хотим удалить файл:

```python
import os

os.remove("file.txt") # Если файл пустой

os.unlink("file.txt") # Если файл не пустой или пустой (универсал)
```

- Окей, давайте попробуем создать папку (директорию)
```python
import os

if not os.path.exists("files"):
    os.mkdir("files") # Создаст папку

if not.os.path.exists("files/files"):
    os.makedirs("files/files" ) # Создаст все необходимые родительские папки

```

- Удалим теперь директории:

```python
import os
import shutil

if os.path.exists("files"):
    os.rmdir("files") # Удалит если пустая папка

if os.path.exists("files/files"):
    shutil.rmtree("files/files") # Удалит всё содержимое (Мощная вещь, осторожно!!!)

```

- Теперь мы хотим узнать размер файла и папки:

```python
import os

if os.path.exists("file.txt"):
    os.path.getsize("file.txt") # os.path.getsize подходит для файлов и папок

if os.path.exists("files"):
    os.path.getsize("files")
```

### Низкоуровневые функции

- Вы знаете как работать с файлами и директориями, но вам надо знать еще низкоуровневые функции

- Давайте попробуем узнать вашу систему:

```python
import platform

syst = platform.platform()

if syst == 'Windows':
    print("У вас Windows")
elif syst == "Linux":
    print("У вас Linux")
else:
    print("У вас macOS")
```

- Есть похожий трюк, но уже с sys.platform


- Теперь хотим узнать аргументы командной строки:

```python
import sys
print(sys.argv)
```

- Теперь версию Python (не в терминале):

```python
print(sys.version)
```

- Теперь архитектуру:

```python
import platform
print(platform.machine())
```

- Теперь процессор:

```python
import platform
print(platform.processor())
```

- Хотим вызвать завершение программы:

```python

sys.exit(0) # 0 - Успех, 1 - Ошибка
```
