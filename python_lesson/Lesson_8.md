## Начинаем изучение

### Что такое ООП и создание ОПП

- Что такое ООП и зачем оно?

- ООП: это группировка данных и действий в одном месте. 

- Без ООП (Как я показывал) не удобно, а используя ООП вы сэкономите время и строчки кода

- Как создаётся ООП в Python (Теория):

- Сначала создаём наш каркас - это `class`, он как набор инструкция, как чертёж дома, но дом - это объект

- Затем пишем конструктор - это `__init__`, он срабатывает автоматически при создании объекта

- Затем пишем сам объект - это готовый инструмент по чертежу

- У нас еще есть много чего: `self` - я сам; метод - функция внутри класса или в коде `def`; атрибут - переменная внутри класса и наследование - дочерний класс берёт все от родителя

### Пишем код `(Примеры ООП)`

- Создайте файл какой то файл (если его нет) для обучения и напечатайте:

```python
class Human: # Создаем класс (чертёж)
    def __init__(self, name, age): # Создаём наш конструктор
        self.name = name # Атрибут
        self.age = age # Атрибут

    def say_hello(self): # Создаем функцию (Метод)
        print(f'Привет, я {self.name} и мне {self.age} лет')
    
if __name__ == '__main__': # Запуск
    human = Human("Python", 27) # Инициализируем объект и передаем значение атрибутов
    human.say_hello()  # Запуск метода
```

- Надеюсь вам понятно, ведь я комментировал, кстати забыл вам рассказать про комментарии: их Python игнорирует, пишут их используя `#` или `"""`

- И еще вам надо знать: если в конструкторене указали значения переменнок (как в примере) не забывайте при инициализации указать объекты (нужные), но если в конструкторе всё указано, не придётся передовать значения
